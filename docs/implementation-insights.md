# zsh-benchmark 実装時の知見

## 概要

このドキュメントでは、zsh-benchmarkプロジェクトの実装過程で得られた技術的な知見と学びをまとめています。

## 1. プラグインマネージャーの特性

### 1.1 同期的 vs 非同期的インストール

**問題**
- ほとんどのプラグインマネージャーは同期的にプラグインをインストール
- zcometは非同期でバックグラウンドダウンロードを実行
- 単純な測定方法では、zcometが数ミリ秒で完了したように見える

**解決策**
```bash
# zcometのダウンロード完了を監視
while [ $TIMEOUT -lt 180 ]; do 
  COMPLETE=$(find ~/.zcomet/repos -name HEAD 2>/dev/null | wc -l)
  # .git/HEADファイルの数で完了を判定
done
```

**学び**
- 非同期処理の測定には、完了を検出する仕組みが必要
- ファイルシステムの変化を監視するのが確実
- 安定性確認（3秒間変化なし）で誤検出を防ぐ

### 1.2 特殊な初期化が必要なマネージャー

**oh-my-zsh / prezto**
- プラグインがフレームワークに組み込まれている
- 個別のインストール時間測定が困難
- `noInstallSupport: true`フラグで対応

**zcomet**
- 非同期ダウンロードのため特殊な測定が必要
- `skipInstall: true`でGitHub Actionsでの問題を回避

## 2. Docker環境での測定

## 3. 測定精度の向上

### 3.1 統計的測定（hyperfine）

**利点**
- 複数回実行で外れ値を除外
- 標準偏差を含む詳細な統計情報
- JSONエクスポートで自動処理が容易

**実装例**
```typescript
const cmd = `hyperfine --warmup ${warmupRuns} --runs ${runs} \
  --export-json /tmp/${manager.name}-load.json \
  'timeout 10 zsh -ic exit'`;
```

### 3.2 エラーバーの重要性

**実装**
- hyperfineから標準偏差を取得
- チャートのツールチップに表示
- `147.5ms ± 12.3ms`形式で可視化

**学び**
- 測定のばらつきを示すことで信頼性が向上
- 特に遅いマネージャーほどばらつきが大きい

## 4. スコア計算の工夫

### 4.1 スケールの違いへの対応

**問題**
- LoadTime: 100-400ms
- InstallTime: 2000-30000ms
- 単純な加重平均では意味がない

**解決策**
```typescript
// 各メトリクスの平均値で正規化
const loadScore = (result.loadTime || 0) / loadTimeAvg;
const installScore = (result.installTime || 0) / installTimeAvg;
const totalScore = loadScore * 0.8 + installScore * 0.2;
```

**効果**
- 異なるスケールのメトリクスを公平に比較
- 新しいマネージャー追加時も自動調整

## 5. インタラクティブ測定への挑戦

### 5.1 romkatv/zsh-benchからの学び

**重要な測定項目**
- First Prompt Lag: ターミナル起動時の待ち時間
- Command Lag: コマンド実行時の遅延
- Input Lag: キー入力の反応速度

**人間の知覚閾値**
- Command Lag < 10ms: 知覚できない
- Input Lag < 20ms: 知覚できない
- First Prompt Lag < 50ms: 知覚できない

### 5.2 実装の課題

**PTY（疑似端末）の制御**
- expectスクリプトを使用
- Dockerコンテナ内でのインタラクティブセッション
- タイミング測定の精度確保

**簡易実装の選択**
```bash
# scriptコマンドでプロンプト出現を検出
script -q -c 'zsh -l -c "exit"' /dev/null | grep -m1 '[$%>❯]'
```

## 8. インタラクティブ測定の実装完了

### 8.1 実装の詳細
expectスクリプトとフォールバック手法を使用したインタラクティブ測定を実装しました：

**実装内容**
- `interactive-benchmark.ts`: expectスクリプトによる精密測定
- `measure-interactive.sh`: シンプルなシェルスクリプトによる測定
- `benchmark-cli.ts`: `--interactive`フラグの追加
- `generate-readme.ts`: インタラクティブメトリクスの表示追加

**測定項目**
1. **First Prompt Lag**: シェル起動から最初のプロンプト表示まで
2. **First Command Lag**: シェル起動から最初のコマンド実行可能まで
3. **Command Lag**: Enterキー押下から次のプロンプト表示まで
4. **Input Lag**: キー入力から画面表示まで

**評価基準（romkatv/zsh-benchに基づく）**
- 🟢 Imperceptible（知覚できない）
- 🟡 Noticeable（感じられる）
- 🔴 Slow（遅い）
