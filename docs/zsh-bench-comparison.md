# romkatv/zsh-bench ベンチマーク手法の分析

## 概要

romkatv/zsh-benchは、インタラクティブなzshのユーザー体感速度を測定するツールです。
私たちのzsh-benchmarkプロジェクトとは異なるアプローチを採用しています。

## 主要な測定項目

### romkatv/zsh-benchの測定項目

1. **first_prompt_lag_ms**
   - シェル起動から最初のプロンプト表示までの時間
   - ターミナルを開いた時の待ち時間

2. **first_command_lag_ms**
   - シェル起動から最初のコマンド実行開始までの時間
   - すぐにコマンドを打ち始めた場合の実行待ち時間

3. **command_lag_ms**
   - Enterキー押下から次のプロンプト表示までの時間
   - 通常のコマンド実行時の体感速度

4. **input_lag_ms**
   - キー入力から画面表示までの遅延時間
   - タイピング時の反応速度

5. **exit_time_ms**
   - `zsh -lic "exit"` の実行時間
   - romkatv氏は「意味がない」と評価

## 測定方法の特徴

### 仮想TTYアプローチ
- 実際のターミナル環境を仮想的に再現
- キーストロークを送信してシェルの反応を計測
- ユーザーの実際の操作に近い形で測定

### 私たちの手法との違い

| 項目 | romkatv/zsh-bench | 私たちのzsh-benchmark |
|-----|-----------------|------------------|
| 測定対象 | インタラクティブな遅延 | プラグインマネージャーの性能 |
| 測定方法 | 仮想TTY + キーストローク | hyperfineによる統計的測定 |
| フォーカス | ユーザー体感速度 | インストール・ロード時間 |
| プラグイン数 | 固定（設定による） | 0, 5, 10, 15, 25で段階的測定 |

## 人間の知覚閾値

romkatv氏による実験結果：

| レイテンシタイプ | 知覚できない最大値 | 備考 |
|--------------|----------:|-----|
| first prompt lag | 50ms | ターミナルを開いた時 |
| first command lag | 150ms | 最初のコマンド実行 |
| command lag | 10ms | 通常のコマンド実行 |
| input lag | 20ms | キー入力の反応 |

これらの値は個人差があるが、大まかな目安として有用。

## 私たちのプロジェクトへの適用可能性

### 取り入れるべき点

1. **ユーザー体感速度の測定**
   - 現在はインストール・ロード時間のみ
   - インタラクティブな遅延も重要

2. **知覚閾値の考慮**
   - 絶対値だけでなく、人間が感じる速度も評価
   - スコア計算に閾値を組み込む可能性

3. **実環境に近い測定**
   - hyperfineは統計的だが、実際の使用感とは異なる可能性
   - TTYベースの測定も検討価値あり

### 現在の制限事項

1. **測定環境の違い**
   - Dockerコンテナ内での測定
   - 実際のターミナル環境とは異なる

2. **フォーカスの違い**
   - プラグインマネージャーの比較が主目的
   - インタラクティブ性能は副次的

## 結論

romkatv/zsh-benchは、ユーザー体験に直結する遅延を測定する優れたツールです。
私たちのプロジェクトも、将来的にはこれらの測定項目を追加することで、
より包括的なベンチマークツールに進化できる可能性があります。

特に、プラグインマネージャーの選択がインタラクティブ性能に与える影響を
測定できれば、より実用的な比較が可能になるでしょう。
